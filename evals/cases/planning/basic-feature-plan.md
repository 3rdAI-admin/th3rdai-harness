# Eval Case: Basic Feature Plan

## Purpose

Test whether the Planner Agent can turn a feature request into an implementation-ready plan.

## Input

Add a new `/health` endpoint to an existing API. The endpoint should return status, version, and uptime. Include tests and documentation.

## Expected Qualities

- Clear goal and scope
- Likely files to inspect before implementation
- Concrete implementation steps
- Specific validation criteria
- Risks around uptime calculation and version source
- Handoff to Builder Agent

## Rubric

Use `evals/rubrics/plan-quality.md`.

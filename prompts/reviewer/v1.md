# Reviewer Prompt v1

## Purpose

Review plans, implementations, prompts, skills, or agent contracts for correctness, completeness, safety, and maintainability.

## Instructions

1. Identify the stated goal and success criteria.
2. Compare the artifact against the goal.
3. Check for missing context, weak validation, unsafe assumptions, and unclear handoffs.
4. Classify findings by severity.
5. Distinguish blocking issues from recommendations.
6. Provide an approval, revision, or rejection recommendation.

## Required Output Sections

- Review Scope
- Summary Judgment
- Blocking Issues
- Recommendations
- Validation Gaps
- Safety Concerns
- Approval Status

## Known Failure Modes

- Reviews can over-focus on style instead of correctness.
- Recommendations can be vague unless tied to exact files or sections.
- Approval can be premature if validation evidence is missing.

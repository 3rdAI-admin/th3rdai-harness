## Risks and Edge Cases

### Risk 1: GitNexus Not Available
**Scenario:** User has MCP server not running or repo not indexed
**Mitigation:** Skill procedure should check availability first, fail gracefully with clear error
**Severity:** Low (user can fix by running `npx gitnexus analyze`)

### Risk 2: Mixed File Types
**Scenario:** Cleanup batch includes code + docs + configs
**Mitigation:** Hybrid approach documented in procedure (GitNexus for code, git grep for others)
**Severity:** Low (clear guidance in procedure)

### Risk 3: False Positives on Risk Level
**Scenario:** GitNexus reports HIGH risk on safe move (unlikely but possible)
**Mitigation:** User can review blast radius and override if confident
**Severity:** Low (better safe than sorry)

### Risk 4: Breaking Change for Early Adopters
**Scenario:** Someone used v1.0.0 template and expects old behavior
**Mitigation:** This is an enhancement, not a breaking change. Old guidance still works, new is better.
**Severity:** None
